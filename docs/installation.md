# Добавление MercuryBot на сервер

## Шаг 1. Приглашение бота

Перейдите по ссылке-приглашению MercuryBot на ваш Discord-сервер:

::: callout tip "Ссылка на приглашение"
Добавьте бота через официальный Discord OAuth2 портал. Убедитесь, что у вас есть права **Управлять сервером** (Manage Server) на сервере, куда добавляете бота.
:::

1. Откройте ссылку приглашения бота в браузере.
2. В выпадающем списке выберите сервер, на который хотите добавить бота.
3. Нажмите **Продолжить**.
4. Оставьте все предложенные права (рекомендуется) или настройте минимально необходимые.
5. Нажмите **Авторизовать**.
6. Пройдите проверку CAPTCHA.

---

## Шаг 2. Необходимые права

Для корректной работы MercuryBot требуются следующие права на сервере:

| Право | Назначение |
|---|---|
| **Управлять ролями** | Выдача ролей из магазина, временные роли |
| **Управлять каналами** | Приватные комнаты, любовные комнаты |
| **Управлять сообщениями** | Очистка чата, авто-модерация |
| **Банить участников** | Команда `/ban` |
| **Кикать участников** | Авто-наказания |
| **Отправлять сообщения** | Ответы на команды |
| **Встраивать ссылки** | Embed-сообщения |
| **Читать историю сообщений** | Модерация, логи |
| **Подключаться к голосовым каналам** | Приватные комнаты, голосовая активность |

::: callout warning "Важно"
Убедитесь, что роль бота находится **выше** ролей, которые он должен выдавать (магазин ролей, временные роли). Иначе бот не сможет выдать роль пользователю.
:::

## Шаг 3. Первичная настройка

После добавления бота на сервер выполните базовую настройку:

### 1. Настройка каналов

Используйте команду `/set` для настройки каналов:

- `/set notifychannel` — канал для оповещений бота
- `/set reportchannel` — канал для приёма жалоб на пользователей

### 2. Настройка логирования

Включите логирование действий на сервере:

- `/set logs` — выберите тип логов (роли, каналы, сообщения, наказания) и укажите канал

### 3. Настройка экономики

- `/set leveling` — настройте систему уровней (множитель опыта, награды)
- `/set systems` — включите нужные системы (экономика, банк, казино и т.д.)

### 4. Настройка модерации

- `/set warn` — установите максимальное количество варнов
- `/set autopunish` — настройте авто-наказания за превышение лимита варнов

---

## Шаг 4. Настройка приватных комнат

Для работы приватных голосовых каналов:

1. Создайте категорию для приватных комнат.
2. Создайте текстовый канал для управления приватами.
3. Выполните `/set private` и укажите:
   - **Категория** — куда будут создаваться каналы
   - **Канал создания приватов** — голосовой канал, при заходе в который создаётся комната
   - **Канал управления приватами** — текстовый канал с кнопкой управления
4. Отправьте панель управления через `/send-private`.

---

## Шаг 5. Настройка тикетов

Для системы поддержки через тикеты:

1. `/set ticket` — настройте:
   - **Тикет канал** — канал, где будет панель создания тикетов
   - **Роль агента поддержки** — роль, которая будет добавляться в тикет
   - **Текст сообщения** — текст в панели тикетов
2. Отправьте панель тикетов через `/send-ticket`.

---

## Шаг 6. Настройка магазина ролей

1. Добавьте роли в магазин: `/add-shop-item`
   - Укажите роль, цену, количество покупок и название
2. Пользователи могут просматривать магазин через `/shop`
3. Покупка роли: `/buy-shop-item`

## Частые проблемы

### Бот не реагирует на команды

- Убедитесь, что бот имеет право **Отправлять сообщения** в канале
- Проверьте, что бот онлайн (зелёный статус)
- Убедитесь, что команды не отключены через `/set systems`

### Бот не выдаёт роли

- Проверьте, что роль бота **выше** выдаваемой роли в списке ролей сервера
- Убедитесь, что у бота есть право **Управлять ролями**

### Бот не создаёт приватные комнаты

- Проверьте, что категория и каналы правильно настроены через `/set private`
- Убедитесь, что у бота есть право **Управлять каналами**
